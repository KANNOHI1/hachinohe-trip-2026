あなたはフロントエンド開発者兼情報設計者です。
GitHub Pagesで公開できる、2026年9月の「八戸・三沢旅行」の旅のしおりWEBサイトを作ってください。

# 目的

2人で共有する旅行計画サイトを作る。
Google Keepに書いているラフなメモを元に、見やすく、あとから更新しやすいWEBサイトへ移行する。

この時点のGoogle Keepには誤記、重複、未確定情報が含まれている。
そのため、Keepの内容をそのまま確定情報として扱わず、以下の方針で整理する。

・原文メモは raw data として尊重する
・明らかな誤記や重複は、サイト側では補正・統合する
・未確定情報には status を付ける
・事実確認が必要なものは「要確認」として表示する
・旅行中にスマホで見やすいことを最優先にする

# 技術要件

・Astro + TypeScript で作成する
・GitHub Pagesで公開できる構成にする
・スマホファーストのレスポンシブデザイン
・外部APIは使わない
・地図はGoogle Mapsへの外部リンクでよい
・データは src/data/trip.ts に分離する
・UIはカード、タイムライン、ステータスバッジを使う
・デザインは「青森、海、温泉、森、旅のしおり」を感じる落ち着いた雰囲気
・まずはMVPとして静的サイトを完成させる
・あとから写真、メモ、予約情報、リンクを追加しやすい構成にする

# ステータス設計

各情報には status を持たせる。

・confirmed：確定
・tentative：仮
・check：要確認
・unavailable：現時点で不可または空室なし
・interest：個人的興味
